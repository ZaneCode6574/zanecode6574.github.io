4.1 线性表

  • 线性表：由称为元素的数据项组成的一种有限且有序的序列。

    • 有序不是指按大小排序，而是每个数据项都有自己的位置
    • 概念上不反对线性表有不同的数据类型的元素，但是一般来说只有一种

  • 不包含任何元素称为空表（empty list），元素数目称为长度（length），开始节点称为表头（head），结尾节点称为表尾（tail）

  • 元素位置可以与大小有关也可以无关。有序线性表（sorted list）元素按照值的递增顺序排列，无序线性表（unsorted list）元素值和位置没有关系

  • C++中下标从0开始

  • 线性表抽象声明

4.1.1 顺序表的实现

  • AList
  • 在商用系统中，最好使用C++的异常处理机制来替换Assert
  • 这种AList中对于增加或者移除，会影响多个元素。如果只是简单的访问元素或者移动栅栏的位置（currPos）代价都是Θ(1)

4.1.2 链表

第二种实现线性表的方法，利用指针，称为链表。链表是动态的，能够按照需要为表中新的元素分配存储空间，

链表由一系列称为表的节点的对象组成的。因为节点是一个独立的对象（这一点与数组的一个元素相反），所以它能很好地实现独立的节点类，还能被栈和队列的链接实现方式重用。

Link中包含一个存储元素值的element，和一个指向表中下一个节点指针的next域，这种结点建立的链表中，每个节点只有指向下一个节点的指针，所以称为单链表或者单向表

  • 实际上Link类应该作为链表实现的一个私有类来实现（这里是为了后面能继续用）

  • 使用head保存链表第一个结点指针，用tail保存最后一个链，用curr保存栅栏的位置，线性表的长度单独保存为cnt，对链表的每个修改都要更新这个值

  • 但是当curr指向当前节点，在插入时会导致没有一种方便的方法指向curr所指节点的前驱节点，因此，将curr指向当前位置的前一个元素就可以解决这个问题

  • 当链表为空或者当前位置已经在末端的时候，会出现一些特殊情况。尤其是当链表为空，没有元素可以供head、tail、curr来指向，当链表左边为空时，curr无法指向任何元素。一种方法时增加一些用于处理特殊情况的代码，另外一种方法是增加特殊的表头节点（header node），表头节点与其他节点一样，只是值被忽略，与第一种方法相比，开销较小，更简单

  • 使用表头节点初始化一个空链表时，head，curr，tail都指向这个位置

(curr是指向的是栅栏前面的节点，而栅栏后面的节点称为当前节点，把老子迷惑得)

  • 插入一个节点实际上有三步

    • 创建一个新的节点
    • 使curr的next指向插入的节点
    • 将curr的next赋值给插入节点的next

  • 删除一个节点只需要重定向被删除节点周围的指针。

    • 比较特殊的就是，在单链表中没有指向前驱节点的指针，成员函数prev把curr指针向表头移动一个位置需要从表头开始沿着链表移动，需要Θ(n)的时间
    • 内存应该返回给存储器，首先将要删除的指针指向临时指针ltemp，然后调用delete函数释放被删除节点占用的内存

可利用空间表

在链表创建和删除节点的时候，Link类的程序员能够提供简单而有效的内存管理例程，以代替系统级的存储分配和回收操作符（new和delete，而且这两个比较难用）。

可利用空间表用来存放当前那些不用的线性表的节点，从一个链表中删除节点就可以放到可利用空间表的首端，当需要把一个新元素增加到链表中时，先检查可利用空间表，看看是否有可用的线性表节点，如果有空节点，则可从可利用空间表中取走一个节点，当可利用空间表为空的时候才会调用标准操作符new

  • 实现方法：
    • 第一种在Link类实现中增加两个函数，代替标准的new和delete
    • 第二种在Link类的对象上进行操作时，用C++操作符重载替代new和delete。在这种方法下，使用LList类的程序不必做任何修改就可以享受到便利。

使用可利用空间表的Link类实现

为了防止需要很多个链表节点的时候不断的调用::new操作，可以一次new多个节点空间出来，毕竟调用一次创建100个节点动作要比调用100次创建1个节点动作来得快

4.1.3 对线性表实现方法的比较

顺序表对于每一个元素都没有空间的浪费，链表在空间上必须分配一部分给结构性开销。设n为线性表中当前元素的数目，P表示的指针的存储单元的大小（通常为4字节），E表示数据元素的存储单元大小（可任意取值，最小为一位的布尔变量，最大可以是几千字节的复杂记录类型），D表示可以在数组中存储的线性表元素的最大数目，则顺序表的空间需求为DE，如果不考虑任何给定时刻链表中实际存储元素的数目，则链表的空间需求为n(P+E)。对于给定的n值，这两个表达式中较小的那一个对于n个元素的实现有着更高的空间效率。一般情况下，当线性表中的元素相对较少时，链表的实现比顺序表的实现更节省空间，反之，当数组几乎填满时，顺序实现方法空间效率更高。

(L^R)^R=L, (L^R)^L=R

template class Stack{
private:
void operator =(const Stack&){}
Stack(const Stack&){}

public:
Stack(){}
virtual ~Stack(){}

virtual void clear() = 0;
virtual void push(const E& it) = 0;
virtual E pop() = 0;
virtual const E& topValue() const = 0;
virtual int length() const = 0;

};

template class AStack: public Stack {
private:
int maxSize;
int top; //Index for top element
E *listArray; //Array holding stack elements

public:
AStack(int size = defaultSize) {
maxSize = size;
top = 0;
listArray = new E[size];
}

~AStack() {
    delete[] listArray;
}

void clear() {
    top = 0;
}

void push(const E &it) {
    Assert(top != maxSize, "Stack is full");
    listArray[top++] = it;
}

E pop() {
    Assert(top != 0, "Stack is empty");
    return listArray[--top];
}

const E &topValue() const {
    Assert(top != 0, "Stack is empty");
    return listArray[--top];
}

int length() const{
    return top;
}

};

template class LStack: public Stack{
private:
Link* top; //pointer to first element
int size;

public:
LStack(int sz= defaultSize){
top = NULL;
size = 0;
}

~LStack(){
    clear();
}

void clear(){
    while(top!=NULL){
        Link<E>* temp = top;
        top = top->next;
        delete temp;
    }
    size = 0;
}

void push(const E& it){
    top = new Link<E>(it, top);
    size++;
}

E pop(){
    Assert(top!=NULL, "Stack is empty");
    E it=top->element;
    Link<E>* ltemp = top->next;
    delete top;
    top = ltemp;
    size--;
    return it;
}

const E& topValue() const {
    Assert(top!=0, "Stack is empty");
    return top->element;
}

int length() const {
    return size;
}

};

template class Queue {
private:
void operator =(const Queue&) {} // Protect assignment
Queue(const Queue&) {} // Protect copy constructor

public:
Queue() {} // Default
virtual ~Queue() {} // Base destructor

// Reinitialize the queue. The user is responsible for
// reclaiming the storage used by the queue elements.
virtual void clear() = 0;

// Place an element at the rear of the queue.
// it: The element being enqueued.
virtual void enqueue(const E&) = 0;

// Remove and return element at the front of the queue.
// Return: The element at the front of the queue.
virtual E dequeue() = 0;

// Return: A copy of the front element.
virtual const E& frontValue() const = 0;

// Return: The number of elements in the queue.
virtual int length() const = 0;
};

template class AQueue: public Queue {
private:
int maxSize; // Maximum size of queue
int front; // Index of front element
int rear; // Index of rear element
E *listArray; // Array holding queue elements

public:
AQueue(int size =defaultSize) { // Constructor
// Make list array one position larger for empty slot
maxSize = size+1;
rear = 0; front = 1;
listArray = new E[maxSize];
}

~AQueue() { delete [] listArray; } // Destructor

void clear() { rear = 0; front = 1; } // Reinitialize

void enqueue(const E& it) { // Put “it” in queue
Assert(((rear+2) % maxSize) != front, “Queue is full”);
rear = (rear+1) % maxSize; // Circular increment
listArray[rear] = it;
}

E dequeue() { // Take element out
Assert(length() != 0, “Queue is empty”);
E it = listArray[front];
front = (front+1) % maxSize; // Circular increment
return it;
}

const E& frontValue() const { // Get front value
Assert(length() != 0, “Queue is empty”);
return listArray[front];
}

virtual int length() const // Return length
{ return ((rear+maxSize) - front + 1) % maxSize; }
};

template class LQueue: public Queue {
private:
Link front; // Pointer to front queue node
Link rear; // Pointer to rear queue node
int size; // Number of elements in queue

public:
LQueue(int sz =defaultSize) // Constructor
{ front = rear = new Link(); size = 0; }

~LQueue() { clear(); delete front; } // Destructor

void clear() { // Clear queue
while(front->next != NULL) { // Delete each link node
rear = front;
delete rear;
}
rear = front;
size = 0;
}

void enqueue(const E& it) { // Put element on rear
rear->next = new Link(it, NULL);
rear = rear->next;
size++;
}

E dequeue() { // Remove element from front
Assert(size != 0, “Queue is empty”);
E it = front->next->element; // Store dequeued value
Link* ltemp = front->next; // Hold dequeued link
front->next = ltemp->next; // Advance front
if (rear == ltemp) rear = front; // Dequeue last element
delete ltemp; // Delete link
size –;
return it; // Return element value
}

const E& frontValue() const { // Get front element
Assert(size != 0, “Queue is empty”);
return front->next->element;
}

virtual int length() const { return size; }
};
